Hadoop高可用集群搭建流程

发表于 2017-03-13 | 分类于 hadoop |

在实际工作中整理出hadoop高可用架构的搭建方案和流程，在之前的单个namenode的基础上升级成了了双namenode和双resourcemananger，从而能够保证可以运用到实际的任务处理中。

一：整体方案

现有机器9台，整体的架构如图

二：前期准备工作

1、防火墙的设置

防火墙对内网开放，对外网全部关闭

2、xiao用户创建

为所有的机器创建统计的用户和用户组xiao，方便统一管理。后续将在xiao用户上进行操作

3、hosts文件配置

为使各个机器能够相互的用主机名进行通和配置hadoop的slaves文件，需在每个机器上都保存其它系统的主机名，编辑_etc_hosts文件，加入

然后传到其它机器上

4、ssh免密钥配置

目的是使各个机子能相互的免密访问(在没有配置免密钥登录上，可以通过expect工具，来使用root用户登录到各个机器上执行以上操作)
先在每台机器上生存key

然后将所有机器的key回传到一台namenode上

最后将这台namenode上的authorized_keys，传到其它机器上

测试一次，看是否能够实现相互免密钥登录

5、JDK设置

机器上自带的jdk为openjdk，其缺乏一些java的管理工具，如查看java进程的工具jps，这在后面搭建好集群后，检测集群的进程是否正常启动很有用。因此选择卸载原有的openjdk，改用oraclejdk。

三： ZooKepper配置

搭建高可用的hadoop集群，需要使用到zookeeper，因此先搭建zookeeper。

1、配置JAVA_HOME

在zookeeper的配置文件夹conf下，如果没有java.env 文件，则创建一个，然后陪着JAVA_HOME

2、配置zoo.cfg

加入以下设置，设置格式为server.=:2888:3888
Id是为每一台zookeeper的设置的唯一id标识，不能重复。

然后将默认的数据目录修改为挂载盘

配置完后，将zookeeper分发到其它的需要装zookeeper的机器

3、myid配置

在每台机器的zookeeper数据目录下创建myid文件，并写入对应机器的server.id

四：Hadoop配置

每台机器的hadoop都是一样，因此可以选择在一台机器上配置完毕，发送到其它的机器上。这里选择一台namenode来配置

1、core-site.xml

2、hdfs-site.xml配置

主要涉及到hdfs的namespace和namenode的ha设置，以及数据目录设置

3、yarn-site.xml

主要设计ResourceManager的高可用设置，zookeeper的地址

4、mapred-site.xml

设置mapreduce，可以直接使用默认的

5、slaves

需要在slaves文件加入namenode需要管理datanode的节点名

6、hadoop-env.sh

7、yarn-env.sh

为 I了保险起见，设置yarn的JAVA_HOME

至此hadoop的配置结束，暂未对hadoop的一些参数进行优化。将配置好的hadoop文件分发到其它机器上。
