SASS使用总结

发表于 2016-10-20 | 分类于 大前端 |

SASS是一种CSS的开发工具，提供了许多便利的写法，大大节省了设计者的时间，使得CSS的开发，变得简单和可维护。
运用sass可以减少写繁琐的css样式，更利用css的模块化编写和维护。

sass的安装使用

  1. 安装
    • sass安装需要ruby环境，因此首先安装ruby环境
    • 安装sass: gem install sass

  2. 常用命令

    1. 将scss文件转化为css
      sass test.scss test.css

    2. SASS提供四个编译风格的选项：

      • nested：嵌套缩进的css代码，它是默认值
      • expanded：没有缩进的、扩展的css代码。
      • compact：简洁格式的css代码。
      • compressed：压缩后的css代码。（生产环境）
        sass --style compressed test.sass test.css

    3. 实时监听sass的变动

      • 监听一个文件
        sass --watch input.scss:output.css
      • 监听整个文件夹
        sass --watch app/sass:public/stylesheets

高级语法

  • 继承

    SASS允许一个选择器，继承另一个选择器

  • Mixin

    Mixin有点像C语言的宏（macro），是可以重用的代码块。
    @mixin命令，定义一个代码块

指定参数和缺省值。

  • 颜色函数
    SASS提供了一些内置的颜色函数，以便生成系列颜色

  • 插入文件
    @import命令，用来插入外部文件:
    @import "path/filename.scss";

bootstrap-sass项目解析

bootstrap-sass是由sass写的bootstrap版本
安装bower install bootstrap-sass

  • stylesheets
    • bootstrap 存放scss的文件
      • mixins 重用的代码块
      • _variables.scss 定义的变量
      • _normalize css的格式代码
      • _glyphicons 字体文件
      • 核心scss
      • 组件的scss
      • 需要js的组件的scss
      • Utility 公用的scss
    • _bootstap.scss 包含引导文件

  1. 在laravel中，直接在resource中的assets的app.scss中引入
    @import "node_modules/bootstrap-sass/assets/stylesheets/bootstrap";
    即可将bootstrap引入项目中

  2. 可以备份后修改_bootstap.scss中引入的scss来按需加载

实践总结

  1. 在编写css的时候，思考是否其它地方也会用到？是否可以进行模块化
  2. 将复杂并且混乱的样式表拆分整理成独立的模块，按需加载。
